// Runs inside RISC Zero zkVM — produces cryptographic receipt
fn main() {
    let trace: ExecutionTrace = risc0_zkvm::guest::env::read();
    let policy: AgentPolicy = risc0_zkvm::guest::env::read();

    // Verify each tool invocation against policy
    let mut all_passed = true;
    for invocation in &trace.tool_invocations {
        if !policy.allowed_tools.contains(&invocation.tool_name) {
            all_passed = false;
        }
    }

    // Determine if Ledger approval is required
    let requires_approval = trace.action_value() > policy.max_value_autonomous;

    // Commit verified output to journal (public)
    risc0_zkvm::guest::env::commit(&output);
}

User Flows

How it works

From agent setup to autonomous execution and Ledger-gated approvals.

Step 01

Connect Ledger & Deploy Agent

Owner connects their Ledger device, picks an ENS subname, and sets policy parameters including allowed tools and spending limits.

Step 02

Agent Receives Task

Via user instruction or encrypted DM3 message from another agent. Intent is classified and routed to the appropriate tool chain.

Step 03

Private Reasoning

Agent reasons via 0G Compute Sealed Inference inside a hardware-isolated TEE. Prompts stay encrypted throughout.

Step 04

Sandboxed Execution

Tools execute in WASM sandbox. Safety layer checks each invocation. Full execution trace is stored on 0G Storage.

Step 05

Generate ZK Proof

Proof request submitted to Boundless. Decentralized provers generate a RISC Zero proof of policy compliance.

Step 06

On-Chain Verification

Proof is verified on-chain. If within autonomous threshold, action executes immediately.

Step 07

Ledger Approval (if needed)

Actions above threshold require physical approval on the owner's Ledger device with human-readable Clear Signing display.

Threat Model

Security

Defense-in-depth across every layer of the stack.

⚠ Agent acts outside policy
RISC Zero proof fails verification; action is blocked on-chain before execution.
⚠ Inference tampering
0G Sealed Inference provides TEE attestation. Response signature is included in the provable trace.
⚠ Message interception
DM3 end-to-end encryption with keys derived from ENS-published profiles.
⚠ Agent identity spoofing
ENS ownership tied to Ledger-controlled EOA. Subname registration requires owner signature.
⚠ High-value action without consent
Ledger physical approval required for all actions above the autonomous threshold.
⚠ Prompt injection
IronClaw safety layer with pattern detection and content sanitization runs inside the proven trace.
⚠ Server wallet compromise
Wallet can only execute actions with valid RISC Zero proofs. High-value actions still require Ledger.

Multi-Agent Token Swap

Alice deploys alice.proofclaw.eth with a policy: swap up to 100 USDC autonomously, above requires Ledger.

Bob deploys bob.proofclaw.eth with a similar policy configuration.

Alice's agent discovers Bob via ENS, initiates DM3 encrypted negotiation.

They agree on a 500 USDC swap — above both autonomous thresholds.

Both agents submit proof requests to Boundless. Provers generate RISC Zero proofs.

Both owners see pending approval on Ledger with Clear Signing. Both press approve.

Swap executes on-chain with verified proofs. Full provability end-to-end.
